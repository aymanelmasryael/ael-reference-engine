# AEL Digital Learning Platform Manifesto

> Internal principles that guide the platform's evolution.

## 1. Technology outlives products.

The framework will outlast any single product built on it.

## 2. Products consume the framework.

Products use the framework; they don't modify it.

## 3. Assets are reusable.

References, curricula, plugins, themes, and certifications are shared across products.

## 4. Plugins extend behavior.

New features are plugins, not core changes.

## 5. Themes customize presentation.

Appearance is a layer, not a modification.

## 6. Data is independent from the engine.

Content lives in data.json or Markdown. The engine renders it.

## 7. Specifications are the source of truth.

The spec defines the standard. The code implements it.

## 8. Community extends the ecosystem.

Contributors build plugins, themes, and references. The core team maintains the framework.

## 9. Backward compatibility is the default.

Breaking changes require major versions and migration guides.

## 10. The Core evolves only in major versions.

Minor versions add features via plugins. Major versions change the core.

---

## Strategic Principle

**Start with one product. Prove the ecosystem. Then expand.**

AEL Engineering Academy is the flagship product. It covers:

- LLM Engineering
- Terminal Engineering
- Git Engineering
- Docker Engineering
- Linux Engineering
- Python Engineering
- JavaScript Engineering
- CSS Engineering

If successful, Design Academy and AI Academy become business decisions, not architectural experiments.
